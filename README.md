# Project Management REST API

A simple REST API built with **Express.js** for managing projects with basic CRUD operations. The application uses **Supabase (PostgreSQL)** as the backend database.

## Features

- Create a project
- Retrieve all projects
- Retrieve a project by ID
- Update a project
- Delete a project
- Search projects by name
- Filter projects by status
- RESTful API design
- Supabase PostgreSQL integration
- Environment variable configuration
- JSON request/response support

## Tech Stack

- Node.js
- Express.js
- Supabase
- PostgreSQL
- dotenv
- CORS

## Getting Started

### Prerequisites

Make sure you have installed:

- Node.js (v18 or later recommended)
- npm
- A Supabase account

### Install Dependencies

```bash
npm install
```

### Create Supabase Database

Go to [Supabase](https://supabase.com), and sign in or create an account. Create a new project and database. Open the **SQL Editor** inside Supabase and execute  the contents of `/src/database/schemas.sql`.

Create `users` table:

```sql
create table public.users (
  id bigint generated by default as identity not null,
  name character varying null,
  email character varying not null,
  password character varying null,
  created_at timestamp with time zone not null default now(),
  constraint users_pkey primary key (id)
) TABLESPACE pg_default;
```

Create `teams` table:

```sql
create table public.teams (
  id bigint generated by default as identity not null,
  name character varying not null,
  description character varying null,
  created_at timestamp with time zone not null default now(),
  constraint teams_pkey primary key (id)
) TABLESPACE pg_default;
```

Create `team_members` table:

```sql
create table public.team_members (
  team_id bigint not null,
  user_id bigint not null,
  constraint team_members_pkey primary key (team_id, user_id),
  constraint team_members_team_id_fkey foreign KEY (team_id) references teams (id) on update CASCADE on delete CASCADE,
  constraint team_members_user_id_fkey foreign KEY (user_id) references users (id) on update CASCADE on delete CASCADE
) TABLESPACE pg_default;
```

Create `projects` table:

```sql
create table public.projects (
  id bigint generated by default as identity not null,
  name character varying null,
  status character varying not null,
  deadline date not null,
  budget character varying not null,
  team_member bigint not null,
  description character varying null,
  created_at timestamp with time zone not null default now(),
  constraint projects_pkey primary key (id),
  constraint projects_team_member_fkey foreign KEY (team_member) references users (id) on update CASCADE on delete RESTRICT
) TABLESPACE pg_default;
```

### Configure Env Variables

Create a `.env` file from `.env.example`.

```env
PORT=5000

SUPABASE_URL=https://your-project_id.supabase.co
SUPABASE_ANON_KEY=your-anon-key

JWT_SECRET=Yx16xPifahoirw6lKRe4tl3uxyJEZi1xyktyuIdb5aw=
JWT_EXPIRES_IN=7d
```

From the Supabase project settings get the project url and the anonymous public API key, and set the corresponding env variables accoringly. You can also change and set `JWT_SECRET` to a random string generated with the command below:

```bash
openssl rand -base64 32
```

### Seed Database

Seed the database tables with the following command:

```bash
npm run db:seed
```

### Running the Server

```bash
npm start
```

or

```bash
npm run dev
```

## API Endpoints

### Registration

Sign up as a new user with the following call:

```http
POST /api/auth/register
```

Request

```json
{
  "name": "John Doe",
  "email": "john.doe@test.com",
  "password": "password"
}
```

Response

```json
{
  "id": 16,
  "name": "John Doe",
  "email": "john.doe@test.com",
  "created_at": "2026-06-30T05:29:14.729136+00:00"
}
```

### Authentication

Get an auth token for subsequent requests.

```json
{
  "email": "john.doe@test.com",
  "password": "password"
}
```

Response

        "created_at": "2026-06-30T05:29:14.729136+00:00"
    },
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpZCI6MSwiaWF0IjoxNzgyNjc4NzIzLCJleHAiOjE3ODMyODM1MjN9.ZIWP5zwpSbjQqzxIb6RcxDD4Q9fi4CEKOJT95QC8bbI",

If you want to filter projects by `status` and `deadline` you could proceed as follows:

```http
GET /api/projects?filter=status:completed,deadline:2026-05-18
```

The call above retrieves `completed` projects having the `deadline` field set to `2026-05-18`.
